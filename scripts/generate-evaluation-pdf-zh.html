目的：功能檢視、流程說明、Campaign Kit 願景、多參考素材策略、市場定位與行銷話術。

執行摘要。技術底層（fal.ai 的 Nano Banana + Seedance 2.0）已相當穩健（約 8/10）。產品體驗仍偏向「分開做圖、分開做片」，未達「一次過出齊一套廣告素材」（新手成功率約 5/10）。影片參考運鏡與部分圖片路徑在輸入、模式或 prompt 唔夾時會令人失望。最高槓桿方向係 通用 Campaign Kit（一個概念，一套素材）：概念 + 產品（+ 可選參考）→ 多項行銷成品，再配合結構化後製編輯——唔係追求「一鍵 100% 完美」。

2. 現時用戶流程

【設定】模板、文案、風格 ↓ 【圖片步驟】 ├─ AI 宣傳圖 → 1 張產品相 → 1 張 AI 圖 ├─ 跟參考圖概念 → 產品 + 1 張參考廣告圖 → 1 張 AI 圖 └─ 固定紙片版面 → 合成器（字準、版面固定） ↓ 【影片步驟】（圖+片 或 只要片） ├─ 圖片變影片 → 1 張關鍵圖 → 約 8 秒動態（參考 MP4 會被忽略） ├─ 跟參考片概念 → 產品 + 1 條參考 MP4 → Seedance 參考生影片 └─ 多角度 → 產品 + 2–3 張角度相 ↓ 【完成】下載（影片附 BGM）

點解感覺「一張圖一張圖、一條片一條片」

  • 圖片模式、影片模式、設定要分開揀，冇一個 活動簡報（campaign brief） 統籌全部輸出。
  • 「圖+片」預設係：做好圖 → 用嗰張圖做片；上傳咗參考 MP4 但模式係「圖片變影片」時，參考片 完全唔會送入模型。
  • 冇「一套素材」：冇 5–6 張輪播、冇 caption 包、冇「故事帖 + Reels + 主圖」一齊出。
  • 用戶心想：我俾咗概念、產品、參考 — 俾一套廣告我。
    程式心想：先 call 圖 API，再 call 片 API。

3. 點解圖片／影片未達高期望

問題後果
影片路徑用錯揀「圖片變影片」→ 上傳嘅 MP4 被忽略 → 變成關鍵圖慢推鏡
@Image1 用錯用 AI 宣傳圖（平面廣告）而唔係原始產品相 → 模型跟唔到參考片嘅手、運鏡
產品同參考唔夾參考片：穿珠、紫水晶；產品：太陽石 macro → 出完全不同類型鏡頭
參考片超過 15 秒／比例唔同Seedance 只食首段約 15 秒；強制 9:16 vs 參考片 4:5
設定同 prompt 打架模板「Slow Push In」覆蓋「跟 @Video1」（近期程式已部分修正）
只出單一資產真實 SMB 廣告通常係多圖 + 一條片 + 文案
生成後難改Done 步只得下載／重生成，未有事後改字、微調圖
Seedance 參考生影片唔係逐格複製。官方建議寫法：@Image1 跟住 @Video1 嘅動作／運鏡。要產品相、剪短參考片、模式正確先似。唔會複製 IG 介面、人樣、商標字樣。

圖片：幾時做得好

跟參考圖概念（產品 + 參考廣告圖 + 標題副標）可以出乾淨嘅客人好評風格 — 跟概念重新排版，唔係照抄像素。純 AI 宣傳圖有時似原相輕修。

4. 願景：概念 + 產品 → 一套行銷素材（Campaign Kit）

「一個概念，一套素材」（適用任何實物商品，唔止水晶）

輸入：產品相片（可多張）· 概念（一段話或預設）· 可選參考圖／參考片 · 繁體文案
輸出：主圖 · 輪播 5–6 張 · Reels MP4 · caption／hashtag · 可選 SRT · ZIP 打包
成品用咩 AI／工具備註
主圖 9:16Nano Banana Pro（跟參考圖概念）IG 主 post
輪播 5–6 張LLM 分鏡 + 每張用 NB／合成器／Ideogram唔好一次 prompt 出 6 張 — 要規劃再批量
Reels 8–12 秒Seedance 參考片 或 圖生影片最好一張 slide 或參考運鏡
Clean MP4現有流程 + ffmpeg BGM俾 CapCut 後製
文案OpenAI（第 3 階段助手）繁體
概念 + 產品 + 參考 → 規劃器（LLM／規則）：模板、模式、每張 slide 簡報 → 批量生成（可平行） → 預覽格：接受／只重做其中一項 → 匯出 ZIP + CapCut 檢查清單

呢個先係市場差異化：編排一整套 campaign，而唔係多一個單圖 API。

5. 單張上傳 vs 多張參考

Seedance 2.0（fal）上限

輸入上限Prompt 標籤
圖片最多 9 張@Image1 … @Image9（產品、角度）
影片最多 3 條，合計 2–15 秒@Video1 …（運鏡、剪輯、手部動作）
音訊最多 3 個，合計 ≤15 秒@Audio1（可選節奏）

幾時多張會更好

  • 同一產品 2–4 個角度（正面、上手、微距）→ 影片身份更穩。
  • @Image2 做風格板，@Image1 做產品。
  • App 已有「更多產品角度」（API 支援多張 images）。

幾時多張反而差

  • 參考互相矛盾（塗鴉 ref + 大理石靜物 + 唔同貨）。
  • 螢幕錄影含 IG 按鈕。
  • 參考片超 15 秒未喺 CapCut 剪短。

介面 vs API

欄位精靈 UIAPI 能力
產品相1 張1+（多角度 2–3 張）
參考廣告圖1 張1 張（風格）
參考 MP41 條1–3 條（≤15 秒合計）

建議：v1 先 1 產品 + 1 參考圖 + 1 條剪短參考片；v2 再做標籤相簿（產品×4、運鏡參考、風格參考）。

8. 對外可講嘅功能與話術

揀 2–3 句長期用：

  1. 一個概念，一套素材 — 產品 + 參考廣告 → 宣傳圖 + Reels + 文案（路線圖中）
  2. 跟參考片概念，唔係亂 generate — 跟運鏡同版面，換你嘅貨
  3. 出 Clean MP4，喺 CapCut 加字幕 — 唔迫你用 AI 字幕
  4. 為小店而設 — 客人好評、新品、限時優惠（模板包）
  5. AI 起稿，你執字執片 — 改文案再出圖 + 專業工具後製

避免承諾：「參考片 100% 一模一樣」或「只適合水晶」。

9. 建議開發優先次序

約 4 週內（建立信任 + Kit MVP）

次序工作原因
1Campaign Kit 模式 — 概念 + 產品 + 參考 → 統籌圖+片解決「分開做」
2規劃器預設 — 有參考片 → R2V + 原始產品相；有參考圖 → 跟參考圖概念減少用錯路徑
3生成後顯示 Seedance 路徑（reference-to-video vs image-to-video）建立信任
4參考片剪短指引（CapCut 8–12 秒）15 秒 API 限制
5Adjust 步驟：改標題副標 → 重新合成／AI 微調圖期望落差
6官網 3 個完整案例管理期望

4–10 週（差異化）

  • 第 3 階段：AI 助手 — 概念 → 分鏡 + 模式
  • 輪播批量：5–6 張（LLM 規劃 + 每張揀 NB／合成器）
  • 產品相簿 3–4 張 → Seedance @Image1–3
  • 第 4 階段：精靈內後製（SRT、軟字幕）
  • 第 5 階段：儲存專案、重新開啟
  • ZIP：slides + MP4 + caption.txt

更後（SaaS）

登入、點數、Stripe、條款 — ROADMAP 第 6–7 階段。內嵌完整 Canva 級編輯器只喺有付費需求先做。

10. 生成後編輯（期望落差）

AI 第一次唔會 100% 啱。適合本專案嘅分層：

層級做咩等於內置 Canva？
1改繁體文案 → 重新合成／對上一張圖做 AI 微調結構化再出圖
2合成器模板輕量編輯（移文字、貼紙）部分
3片：裁剪、BGM、SRT；運鏡要改 → 重生或 CapCut冇時間軸編輯
4ZIP + CapCut／Canva 教學夥伴工作流
5完整內嵌 Canva唔建議 MVP 做
定位：AI 起稿 → 用 CapCut／Canva 執最後 10%。賣「AI 起稿，你執字執片」，唔賣「一鍵完美」。

輪播 5–6 張可組合嘅 AI

  • LLM — 6 格分鏡 + 每格繁體文案
  • Nano Banana Pro — 產品 + 參考廣告圖
  • 合成器 — 字一定要準嘅格
  • Ideogram（可選）— 大字海報格
  • Seedance 2.0 — 揀最好一張做 Reels 或跟參考片

11. 高品質測試清單

  1. 輸出目標：圖+片 或 Campaign Kit（做好之後）
  2. 影片 @Image1 用原始產品相（2–3 張更好），唔好用 AI 宣傳圖做片嘅產品參考
  3. 參考廣告圖 → 跟參考圖概念
  4. 參考 MP4 喺 CapCut 剪成 8–12 秒精華 → 跟參考片概念
  5. 720p、10–12 秒、fast 關閉（參考片模式）
  6. 生成後睇提示：應有 reference-to-video · 1 ref video
  7. 唔啱就改輸入或等 Adjust 功能再出
